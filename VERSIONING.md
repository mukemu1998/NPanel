# 版本号规则

这份文档用于说明 `NPanel` 开源仓库后续如何管理版本号。

当前 `package.json` 中还是：

```text
0.0.0
```

这表示项目还没有正式开始按开源发布节奏维护版本。

## 建议起步方式

建议从第一次稳定的开源可用版本开始，切到：

```text
0.1.0
```

适合升级到 `0.1.0` 的条件：

- 开源仓库内容已清理干净
- 部署文档、维护文档、FAQ、更新日志已经齐全
- 示例截图和说明可以独立支撑部署
- 基础功能可以正常演示和部署

## 推荐规则

### 1. 补丁版本 `0.1.x`

适用于：

- 修 bug
- 文案和文档纠正
- 小 UI 调整
- 安全依赖小升级
- 不改变使用方式的小修正

### 2. 次版本 `0.x.0`

适用于：

- 增加新功能
- 增加新的导出模式
- 明显调整界面结构
- 追加新的部署方式
- 文档结构或项目结构有阶段性变化

### 3. 主版本 `1.0.0`

适用于：

- 开源项目结构已经稳定
- 部署方式和使用方式基本定型
- 后续主要以迭代维护为主

在那之前，继续用 `0.x.y` 更灵活。

## 推荐配套动作

每次版本变化时，建议一起做三件事：

1. 更新 `package.json` 中的 `version`
2. 在 [CHANGELOG.md](CHANGELOG.md) 中追加本次变化
3. 如果是公开发布，顺手补一个 GitHub Release

## 当前建议

如果准备把 `NPanel` 作为正式公开项目持续维护，比较合适的下一步版本是：

```text
0.1.0
```
